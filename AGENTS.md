# AGENTS.md

## Scope
This file applies to the whole repository.

## Project Intent
`appimage-wrapper` provides a small Bash launcher for versioned AppImage files and a desktop-entry generator that uses it.

## Canonical Paths
- Launcher: `bin/appimage-wrapper`
- Desktop generator: `bin/appimage-desktop`
- Bash completions: `bash/appimage-wrapper`, `bash/appimage-desktop`
- Zsh completions: `zsh/_appimage-wrapper`, `zsh/_appimage-desktop`
- Package definition: `PKGBUILD`

## Guardrails
- Keep runtime scripts Bash-only and dependency-light.
- Do not add compiled build steps.
- Keep desktop entries generated by `appimage-desktop` compatible with the freedesktop desktop-entry format.
- Do not make completions depend on non-standard shell plugins.

## Packaging Rules
- Install commands to `/usr/bin`.
- Install Bash completions to `/usr/share/bash-completion/completions`.
- Install Zsh completions to `/usr/share/zsh/site-functions`.
- Bump `pkgrel` for packaging-only changes.
- Bump `pkgver` for user-visible behavior changes intended for release.

## Local Validation Checklist
Run before handing off changes:

```bash
bash -n bin/appimage-wrapper bin/appimage-desktop bash/appimage-wrapper bash/appimage-desktop
zsh -n zsh/_appimage-wrapper zsh/_appimage-desktop
makepkg -f --nodeps --noconfirm
```

## Non-Goals
- Do not manage AppImage downloads.
- Do not install generated desktop entries globally.
- Do not parse shell syntax in flags or env files; keep the documented line-oriented formats.
